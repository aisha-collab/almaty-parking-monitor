#  Almaty Parking Monitor 

> Интеллектуальная система сбора данных о парковках города Алматы с модулем автоматической валидации и WhatsApp-рассылки уведомлений.

---

## ⚡ Быстрый старт

### 1. Клонирование и установка зависимостей

> Скрипты адаптированы под **Python 3.9+** и не требуют тяжёлых системных компиляторов.

```bash
git clone https://github.com/aisha-collab/almaty-parking-monitor.git
cd almaty-parking-monitor
pip install pandas twilio
```

### 2. Настройка конфигурации

```bash
cp .env.example .env
```

Заполните `.env` своими ключами:

| Переменная | Описание |
|---|---|
| `DGIS_API_KEY` | Ключ Catalog API от [dev.2gis.ru](https://dev.2gis.ru) |
| `GOOGLE_SPREADSHEET_ID` | ID таблицы из URL Google Sheets |
| `TWILIO_ACCOUNT_SID` | SID из панели [console.twilio.com](https://console.twilio.com) |
| `TWILIO_AUTH_TOKEN` | Токен авторизации Twilio |
| `TWILIO_WHATSAPP_NUMBER` | Номер Sandbox (пример: `whatsapp:+14155238886`) |

> 📁 Положите файл `creds.json` (сервисный ключ Google) в корневую директорию проекта.

## 📊 Задача 1 — Парсер 2ГИС

### Проблема и архитектурное решение

API 2ГИС имеет жёсткое серверное ограничение: **не более 10 объектов за один запрос** (при `page_size=50` возвращается `400 Bad Request`).

**Решение:** комбинированный подход пространственного анализа — скрипт разбивает территорию Алматы на **36 пересекающихся радиусных зон** (гео-сетку) и поочерёдно сканирует каждую с `page_size=10`.

### Пайплайн обработки

```
Сканирование 36 зон → 1066 "сырых" объектов
        ↓
Дедупликация на стыках зон + очистка пустых строк
        ↓
212 уникальных парковок → parkings_almaty.csv + Google Sheets
```

### Запуск

```bash
python parser.py
```

**Результат:**
- Генерируется локальный файл `parkings_almaty.csv` (212 строк)
- Данные автоматически заливаются в Google Sheets с очисткой старых записей и форматированием заголовков

### Собираемые поля

| Поле | Описание |
|---|---|
| `id` | ID объекта в 2ГИС |
| `name` | Название парковки |
| `address` | Адрес |
| `lat` / `lon` | Координаты |
| `url_2gis` | Ссылка на объект |
| `district` | Район / Город |

## 🤖 Задача 2 — WhatsApp-рассыльщик

Модуль автоматической валидации номеров и отправки шаблонизированных уведомлений. Логика **симуляции** и **реальной отправки** жёстко разделены.

### Режим 1: Dry-Run (симуляция) — безопасно

Полностью тестирует поведение системы на всей базе данных (212 строк) **без внешних сетевых запросов**.

```bash
python sender.py --parking-csv parkings_almaty.csv --mode dry-run
```

> Скрипт мгновенно валидирует базу, форматирует сообщения и сохраняет детальный отчёт в `dry_run_*.log`.

### Режим 2: Live (реальная отправка)

```bash
python sender.py --parking-csv my_test_number.csv --mode live
```

> ⚠️ Для демонстрации передавайте CSV-файл только с вашим личным тестовым номером.

## 🛡️ Отказоустойчивость и валидация

### Нормализация номеров

Скрипт автоматически приводит любые казахстанские форматы к стандарту **E.164**:

```
8 777 123-45-67  →  +77771234567
7777 123 45 67   →  +77771234567
+7 (777) 123-45-67  →  +77771234567
```

### Изоляция сбоев

| Ситуация | Поведение |
|---|---|
| Невалидный номер или пустое поле | `WARNING` в лог → переход к следующей строке |
| Длина сообщения > 1600 символов | `SKIP` (лимит Twilio API) |
| Ошибка лимитов API / обрыв сети | `ERROR` в лог → безопасный переход без падения скрипта |
